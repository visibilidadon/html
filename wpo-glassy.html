CLS por popup MailerLite — iframe inyectado sin dimensiones reservadas 🔴 Crítico
📄 Afecta: HOME · GSC: 19 URLs fallando · CWV fallo activo
Mejora CLS estimada
−0.15

Diagnóstico: MailerLite inyecta un iframe popup que desplaza el contenido de la página al cargarse después del primer paint. El popup no tiene dimensiones reservadas en el DOM inicial, causando un layout shift masivo.

Solución 1 — Retrasar el popup 3s (Shopify · theme.liquid):

liquid / theme.liquid
{%- comment -%} Retrasar MailerLite popup para no impactar CLS {%- endcomment -%}
<script>
  // Cargar MailerLite sólo después de que el usuario interactúe
  function loadMailerLite() {
    var s = document.createElement('script');
    s.src = 'https://assets.mailerlite.com/js/universal.js';
    s.async = true;
    document.head.appendChild(s);
  }
  // Disparar tras 3s o en primer scroll/click (lo que ocurra antes)
  var timer = setTimeout(loadMailerLite, 3000);
  ['scroll', 'touchstart', 'keydown'].forEach(function(e) {
    window.addEventListener(e, function() {
      clearTimeout(timer);
      loadMailerLite();
    }, { once: true, passive: true });
  });
</script>

Solución 2 — Reservar espacio con CSS (evita layout shift si carga antes):

CSS
/* Reservar espacio para el contenedor MailerLite */
.ml-form-embed, [data-ml-form] {
  min-height: 1px;
  contain: layout;
}
/* Evitar CLS en overlays/modals: usar position:fixed en lugar de modificar el flow */
.ml-form-overlay {
  position: fixed !important;
  top: 0; left: 0;
  width: 100%; height: 100%;
  z-index: 9999;
}

Fuentes web sin font-display: swap — FOIT causa CLS y FCP 🔴 Crítico
📄 Afecta: HOME, Collection, Product
Mejora CLS estimada
−0.05

Diagnóstico: Las fuentes web se cargan sin font-display: swap, provocando FOIT (Flash of Invisible Text) que desaparece y reaparece, generando layout shifts y retrasando el FCP.

CSS · fonts.css o en theme.liquid
/* Añadir font-display:swap a TODAS las declaraciones @font-face */
@font-face {
  font-family: 'NombreFuente';
  src: url('fuente.woff2') format('woff2');
  font-display: swap;  /* ← AÑADIR ESTA LÍNEA */
  font-weight: 400;
  font-style: normal;
}

/* Si usas Google Fonts, añade &display=swap al URL */
<!-- ANTES -->
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;700" rel="stylesheet">
<!-- DESPUÉS -->
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;700&display=swap"
      rel="stylesheet">

GIFs animados de 5+ MB en HOME — deben convertirse a vídeo 🔴 Crítico
📄 Afecta: HOME · Speed Index, LCP
Ahorro transferencia
~4.5 MB

Diagnóstico: Los GIFs son el formato más ineficiente para animaciones. Un GIF de 5 MB puede reducirse a 300–500 KB en formato WebM/MP4.

Solución — Convertir GIF a vídeo con ffmpeg:

Terminal (ffmpeg)
# Convertir GIF a WebM (mejor compresión, ~90% menos peso)
ffmpeg -i animacion.gif -c:v libvpx-vp9 -b:v 0 -crf 33 -pass 1 -an -f webm /dev/null
ffmpeg -i animacion.gif -c:v libvpx-vp9 -b:v 0 -crf 33 -pass 2 -an animacion.webm

# Convertir GIF a MP4 (fallback)
ffmpeg -i animacion.gif -movflags faststart -pix_fmt yuv420p -vf "scale=trunc(iw/2)*2:trunc(ih/2)*2" animacion.mp4

Implementación en Liquid (Shopify):

liquid / section .liquid
<video autoplay loop muted playsinline
       width="600" height="400"
       aria-label="Descripción de la animación">
  <source src="{{ 'animacion.webm' | asset_url }}" type="video/webm">
  <source src="{{ 'animacion.mp4' | asset_url }}" type="video/mp4">
</video>

Ahorro estimado
~8 MB

Diagnóstico: El vídeo de demostración de producto se descarga completamente en la carga inicial aunque no sea visible. 9.9 MB es inaceptable para móviles.

# Comprimir vídeo a H.264 optimizado para web (target ~1-2MB)
ffmpeg -i producto-original.mp4 \

Animaciones hotspot con propiedades no-composite (top/left) — CLS y jank 🟠 Importante
📄 Afecta: HOME · CLS, FPS, layout thrashing
Mejora CLS
−0.03 est.

Diagnóstico: Los botones hotspot interactivos de la HOME usan animaciones CSS que modifican propiedades top, left, width o margin. Estas propiedades fuerzan al navegador a recalcular el layout en cada frame, causando "layout thrashing" y contribuyendo al CLS.

CSS — Migrar animaciones a propiedades composite
/* ❌ ANTES: propiedades que fuerzan layout recalculation */
.hotspot-btn {
  animation: pulse-old 2s infinite;
}
@keyframes pulse-old {
  0%   { width: 20px; height: 20px; }
  50%  { width: 28px; height: 28px; } /* ← fuerza layout */
  100% { width: 20px; height: 20px; }
}

/* ✅ DESPUÉS: solo transform y opacity (GPU-composited, sin layout) */
.hotspot-btn {
  width: 20px;
  height: 20px;
  will-change: transform;  /* hint al navegador para GPU layer */
  animation: pulse-new 2s infinite;
}
@keyframes pulse-new {
  0%   { transform: scale(1);   opacity: 1; }
  50%  { transform: scale(1.4); opacity: .8; } /* ← solo GPU */
  100% { transform: scale(1);   opacity: 1; }
}

CSS no utilizado — 167 KiB HOME, 58 KiB Product 🔵 Menor
📄 Afecta: HOME, Product · FCP, transferencia
Ahorro transferencia
~140 KiB

Diagnóstico: Gran parte del CSS se descarga pero nunca se aplica en la página actual. Esto suele ocurrir cuando el tema Shopify carga un único stylesheet global para todas las plantillas.

  • Usar PurgeCSS o el coverage de Chrome DevTools para identificar reglas sin usar
  • Separar el CSS crítico (above-the-fold) del resto usando la técnica de Critical CSS
  • Cargar el CSS no crítico de forma asíncrona con media="print" + onload
liquid · theme.liquid — Carga asíncrona de CSS no crítico
<!-- CSS crítico inline (above-the-fold) -->
<style>
  /* Pegar aquí el CSS mínimo para renderizar el fold inicial */
  body { font-family: sans-serif; margin: 0; }

Notas específicas para Shopify

⚙️ Limitaciones de la plataforma Shopify:
  • Apps de terceros: Cada app instalada en Shopify puede inyectar scripts globales. Revisar en Admin → Tienda online → Temas → Editar código qué scripts se cargan en theme.liquid.
  • CDN de Shopify: Shopify usa su propia CDN global para assets. Los tiempos de TTFB bajos (0.2–0.6s) confirman que la CDN funciona bien — no es necesario cambiar de hosting.
  • Imágenes: Shopify redimensiona y sirve imágenes en WebP automáticamente si se usan los helpers image_tag o img_url correctamente. No es necesario convertir imágenes manualmente.
  • Critical CSS: En Shopify Online Store 2.0, el CSS se puede segmentar por sección usando {% stylesheet %} dentro de cada archivo de sección para carga más granular.
  • Script de apps: Considerar usar Shopify Web Performance App (oficial) o SpeedBooster Pack para automatizar algunas optimizaciones sin editar código directamente.
